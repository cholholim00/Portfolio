## 📌 프로젝트 개요

| 항목 | 내용 |
| --- | --- |
| **프로젝트명** | BridgeBoard - 감정 분류 AI 기반 소통 분석 플랫폼 |
| **목표** | 한국어 텍스트/음성으로부터 감정을 2단계로 분류하여 소통 패턴 분석 |
| **데이터셋** | AI Hub 감성대화말뭉치 (51,630개 대화 샘플) |

| **현재 성과** | 1단계 대분류 모델 Test Accuracy **70.58%** 달성 |

<br>

## 🧠 감정 분류 체계 (klue/roberta-large)

| 대분류 (1단계) | 통합 소분류 (2단계) | 클래스 수 |
| --- | --- | --- |
| **불안** | 걱정/초조, 두려움, 고립감, 혼란, 불안 | 5개 |
| **분노** | 짜증/스트레스, 분노/격분, 방어/혐오 | 3개 |
| **상처** | 억울/배신, 상처/충격, 고립/외로움 | 3개 |
| **슬픔** | 우울/절망, 슬픔/눈물, 후회/실망, 고통 | 4개 |
| **당황** | 부끄러움, 죄책감/환멸, 열등/질투 | 3개 |
| **기쁨** | 감사/신뢰, 만족/편안, 신남/흥분, 자신감 | 4개 |

## 🔄 2단계: 소분류 모델 개발 (진행 중)

### 💡 설계 방식
* 대분류별 전용 모델 6개 각각 학습
* 사람문장 분리로 데이터 3배 확보 (클래스당 ~600개 → ~1,800개)
* 비슷한 소분류 통합으로 클래스 수 축소
* Focal Loss (gamma=2.0) + 클래스 가중치 적용

## 🏗 시스템 아키텍처 (Data Flow)

```text
[ 사용자 ]
  ├── 텍스트 입력 / 음성 입력 (Web Speech API)
  ↓
[ React 프론트엔드 ]
  ├── UI 렌더링 / 감정 결과 시각화
  ↕ REST API 통신
[ FastAPI 백엔드 ]
  ├── 요청 처리 / 모델 추론 / 응답 반환
  ↓
[ 2단계 AI 감정 분류 파이프라인 ]
  ├── 1단계: 대분류 예측 (불안/분노/상처/슬픔/당황/기쁨)
  ↓
  └── 2단계: 소분류 예측 (대분류별 전용 파생 모델 6개)
  ↓
[ 최종 출력 ]
  └── 대분류 라벨 + 소분류 라벨 + 모델 신뢰도 점수(Confidence)
