 BridgeBoard AI 서버 모델 학습 및 디렉토리 구조 명세서
===================================================================

[1. 기본 인프라 및 사전학습 모델 (Base Backbone)]
-------------------------------------------------------------------
* 베이스 모델: klue/roberta-large [cite: 55]
  - 한국어 대규모 텍스트로 사전학습되어 문맥 이해 및 자연어 처리 성능 최적화 [cite: 55]
* 작업 디렉토리: ai_server/


[2. 디렉토리 내부 폴더 및 파일 구조 (Directory Structure)]
-------------------------------------------------------------------
├── data/                  # 원천 데이터 관리 폴더
├── dataset/               # 전처리 완료된 학습용 데이터셋 저장 폴더
├── model_stage1/          # 1단계 대분류 모델 파일(가중치/설정) 저장 폴더
├── model_stage2/          # 2단계 소분류 모델 파일(가중치/설정) 저장 폴더
├── prepare_data.py        # 1단계 대분류 데이터 전처리 스크립트
├── train_model.py         # 1단계 대분류 모델 학습 스크립트
├── prepare_data_stage2.py # 2단계 소분류 데이터 전처리 스크립트
├── train_model_stage2.py  # 2단계 소분류 모델 학습 스크립트
└── evaluate_model.py      # 최종 모델 성능 평가지표 산출 스크립트


[3. 2단계 계층적 감정 분류 파이프라인 (Hierarchical Classification)]
-------------------------------------------------------------------
* 전체 프로세스 개요: 데이터 준비 -> 모델 학습 -> 모델 저장 -> 저장된 모델 로드 [cite: 55]

-------------------------------------------------------------------
 ▶ [STAGE 1] 대분류 예측 아키텍처 (기본 6대 감정 분류 축 구축) [cite: 55]
-------------------------------------------------------------------
1. 대분류 데이터 준비 및 정제
   - 실행 파일: ai_server/prepare_data.py [cite: 55]
   - 주요 기능: 데이터셋을 대분류 6개 라벨(불안/분노/상처/슬픔/당황/기쁨)로 정제 및 전처리 [cite: 55]

2. 대분류 모델 Fine-tuning 학습
   - 실행 파일: ai_server/train_model.py [cite: 55]
   - 주요 기능: 6대 대분류 감정을 높은 정확도로 1차 분류해내는 인공지능 모델 학습 [cite: 55]
   - 결과 저장: model_stage1/ 폴더로 학습 가중치 자동 persist [cite: 55, 56]

-------------------------------------------------------------------
 ▶ [STAGE 2] 소분류 예측 아키텍처 (감정 축 내부 문맥 세분화) [cite: 55]
-------------------------------------------------------------------
3. 소분류 데이터 준비 및 파티셔닝
   - 실행 파일: ai_server/prepare_data_stage2.py [cite: 55]
   - 주요 기능: 1단계 대분류 세션 내에서 하위 미세 감정을 분류하기 위한 데이터 전처리 [cite: 55]

4. 소분류 모델 Fine-tuning 학습
   - 실행 파일: ai_server/train_model_stage2.py [cite: 55]
   - 주요 기능: 각 대분류별 세부 맥락을 정밀 분류하는 소분류 6개 파생 모델 구축 [cite: 55]
   - 결과 저장: model_stage2/ 폴더로 최종 가중치 세트 저장 [cite: 55, 56]


[4. 인프라 서빙 및 사후 관리 유틸리티 (Core Utilities)]
-------------------------------------------------------------------
5. 모델 가중치 영속화 및 체크포인트 세이브 (Model Save)
   - 실행 파일: ai_server/save_model.py (또는 train 스크립트 내부 빌트인) [cite: 56]
   - 주요 기능: 최적의 성능을 낸 에포크(Epoch)의 가중치를 독립적으로 디스크에 아카이빙 [cite: 55, 56]

6. 가중치 로드 및 인퍼런스 엔진 연동 (Model Load)
   - 실행 파일: ai_server/load_model.py (또는 FastAPI 초기화 세션 내 내장) [cite: 56]
   - 주요 기능: 가중치가 완료된 바이너리 파일을 불러와 실시간 추론 파이프라인에 결합 [cite: 56]

7. 테스트셋 기반 모델 정량 평가 및 검증 (Model Evaluation)
   - 실행 파일: ai_server/evaluate_model.py [cite: 56]
   - 주요 기능: 검증 데이터셋을 활용해 학습 완료된 모델의 Accuracy, F1-Score 정량 지표 산출 [cite: 56]
